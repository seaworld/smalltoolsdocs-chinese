Hello, World!

让我们实现一个很小的应用用来展示信息。

创建一个简单request handler

创建一个目录 helloworld. 所有的应用文件都在这个目录。

在 helloworld 目录里, 创建一个文件 helloworld.py, 给它下面的内容： 

import webapp2

class MainPage(webapp2.RequestHandler):
  def get(self):
      self.response.headers['Content-Type'] = 'text/plain'
      self.response.out.write('Hello, webapp World!')

app = webapp2.WSGIApplication([('/', MainPage)],
                              debug=True)

python脚本响应一个htpp request 请求头，描述内容信息 Hello, world!.

创建配置文件

appengine应用有一个 app.yaml的配置文件，除其他事情外，文件描述那个处理脚本使用那个URLS。

在目录helloworld里，创建一个  app.yaml，加上下列内容：

application: helloworld
version: 1
runtime: python27
api_version: 1
threadsafe: true

handlers:
- url: /.*
  script: helloworld.app

从上到下，这个配置文件表达下面的内容：

  • 应用的表示是  helloworld.当你注册应用，你会选择一个独一无二的标识，更新它，在开发的时候可以试任意值，现在，我们设置它为helloworld。
  • 这个版本号 1 是应用的代码，如果你上传了新的版本，appengine会保留先前的一个版本，在管理界面可以让你回滚之间的版本。
  • 这个代码运行在python27下，版本1，附带着运行环境和语言也将在以后被支持。
  • 这个应用是  threadsafe ，所以示例能处理几个同时的request，线程安全是一个高级特性，在你的应用不是特别设计线程安全的话，可能会导致性能不稳定。
  • 每个请求的URL都会匹配正则  /.* ，都会让  helloworld 模块的app对象处理。

这个文件遵循 YAML.语法，对于一个完整的选项列表，看 the app.yaml reference.

测试应用

对处理脚本和配置文件，映射url和处理器，应用就完成了，你可以现在在本地的appengine sdk里测试它。

如果你使用GAE运行，你能设置应用选择菜单，添加应用... 然后选择helloworld目录，选择app的列表，开始，点击浏览按钮来展示它，点击浏览，就会简单的加载 http://localhost:8080/ 在你的默认web浏览器。

如果你不用GAE启动器，在命令行中启动web服务器，进入helloworld目录：

 

google_appengine/dev_appserver.py helloworld/

web服务器现在运行了，监听8080，你可以访问下面的url来测试应用：

  • http://localhost:8080/

关于更多的信息关于开发服务器，包括修改端口，看the Dev Web Server reference, 或者在命令中输入--help.

迭代开发

你可以在开发应用的时候不关心web服务器的运行，web服务器知道修改的部门文件，会在正确的情况下重启。

现在试试：离开web运行服务器，修改 helloworld.py 中  Hello, world! 成别的什么，刷新 http://localhost:8080/ 或点击GAE启动器的按钮来查看改动。

关闭web服务器，确保终端窗口是活动的，按下Ctrl-c(或者是控制台中适当的终端按钮)，或者点击GAE启动器的stop。
